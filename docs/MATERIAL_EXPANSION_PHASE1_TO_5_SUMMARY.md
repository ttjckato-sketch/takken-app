## 2. 追加済み教材（Input Unit）一覧

### 第1弾：宅建業法（金・契約・報酬）
- **営業保証金 / 弁済業務保証金分担金**: 供託先と金額の対比。
- **媒介契約3形態比較**: 一般・専任・専属専任の法的義務の差。
- **報酬額制限**: 計算基礎と2024年最新改定への対応。

### 第2弾：宅建業法（主体・制限）
- **免許制度**: 大臣免許 vs 知事免許、有効期間、免許換え。
- **宅建士 / 専任宅建士**: 設置基準 5:1、独占業務、登録と交付の区別。
- **8種制限 / 自ら売主制限**: 業者・素人区分の全体像、業者間適用除外。

### 第3弾：宅建業法（書面・実務）
- **35条重要事項説明 / 水害ハザードマップ**: 2020年義務化、所在地提示。
- **建物状況調査 / インスペクション**: 媒介・35条・37条の3段階フロー。
- **37条書面 / 必要的・任意的記載事項**: 契約内容の確定と交付義務。

### 第4弾：法令上の制限（基本）
- **都市計画法 / 開発許可**: 区域ごとの面積要件、調整区域の抑制性。
- **建築基準法 / 道路・接道義務**: 42条道路定義、セットバック、2m接道。
- **用途地域 / 建ぺい率・容積率**: 形態規制の計算と緩和条件。

### 第5弾：法令上の制限（応用）
- **農地法 / 3条・4条・5条**: 権利移動と転用の区分、調整区域の厳格審査。
- **盛土規制法（2023年改正対応）**: 土地用途不問の包括規制、2つの規制区域。
- **土地区画整理法 / 仮換地・換地処分**: 事業の時系列、権利移転の「翌日」ルール。

## 3. 重要な修正・法的整合性履歴
- **報酬額制限**: 2024年7月改定（物件価格800万円以下・報酬税込33万円上限）を完全反映。
- **開発許可**: 市街化調整区域を「原則開発抑制」の区域として再定義。
- **農地法**: 調整区域を「原則許可制・厳格審査」として、安易な転用可能性の誤認を排除。
- **法的根拠**: 業法31条の3（専任宅建士）、業法78条2項（業者間免除）など、条文番号を精緻化。

## 4. 技術的改善と検証効率化
- **タグ一致数スコアリング方式**: 複数タグを持つカードに対し、最も具体的な解説ユニットを優先表示する知的なマッチングを実現。
- **dev-only RepairPreview 導線**: 
    - `import.meta.env.DEV` による開発者限定 UI。
    - URLパラメータ `?devRepairTag=` によるダイレクト表示機能。
    - Dashboard下部の「検証用クイックパネル」の実装。
    - **成果**: Antigravityの実測回数を削減し、ランダム出題を待たずに瞬時の目視確認が可能に。

## 5. 今後の運用・検証方針
### Antigravity（実ブラウザ実測）の利用条件
今後はリソース節約のため、以下の「節目」のみで使用する。
- 3〜5論点の新規追加が完了したとき。
- 表示ロジック（Reactコンポーネント等）に大きな変更を加えたとき。
- ビルドは通るが、実ブラウザでのレンダリングに不安があるとき。
- リリース候補（RC）の最終署名前。

### 日常的な確認方法
- `npm run build` による型チェックとビルド整合性。
- 特定タグを指定した URL による `dev-only preview` での目視確認。

## 6. 安全性と禁止事項
### 維持される安全性
- DBスキーマの安定性（既存データを壊さない）。
- 不完全データ（`recovery_pending`, `broken` 等）の出題除外フィルタの継続。
- 学習履歴（`study_events`）の正確な永続化。

### 今後やらないこと
- 学習目的以外のUI・ダッシュボードの過度な装飾。
- 準備不足な論点の無理な〇×化出題。
- DBの物理削除を前提とした不具合回避。
