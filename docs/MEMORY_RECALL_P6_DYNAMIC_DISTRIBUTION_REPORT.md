# MemoryRecall P6 Dynamic Distribution Report

## 概要
FSRS の実測データと学習パフォーマンス（正答率、復習期限切れ数等）に基づき、Daily Session の最適な配分を算出する「動的配分ロジック」を実装した。
本フェーズでは安全性を重視し、算出した推奨配分を `db-audit.html` で確認できる「可視化」までを完了し、実際の自動適用は OFF（24:6 固定）を維持している。

## 実施内容
- **動的指標算出の実装**: `analytics.ts` に `calculateDynamicDistributionMetrics` を追加。直近7日の Active/Memory 正答率、FSRS Due数、レーティング分布、セッション完走率を算出。
- **推奨配分アルゴリズムの実装**: `calculateRecommendedModeDistribution` を追加。安全制約（Active 最低 22問）を守りつつ、弱点補強や復習負荷に応じた配分を試算。
- **安全ガードレールの統合**: 正答率低下時の演習重点化や、消化不良（低いRating）時の増強停止などのガードロジックを組み込み。
- **監査画面の拡張**: `db-audit.html` に「P6 動的配分 推奨」セクションを追加。推奨配分とその根拠となる各指標、および安全ガードの発動状況をリアルタイムで表示。

## 結果 (実測値)
- **算出指標**: Active/Memory 正答率(7d), FSRS Due数, Overdue数, Rating分布, 未完了日数
- **推奨配分範囲**: Active 22-26 : Memory 4-8 (常に Total 30)
- **自動適用フラグ**: `auto_apply: false` (P6 規定通り OFF 維持)
- **Build 結果**: PASS (TypeScript / Vite)
- **Schema 整合性**: FULL_SYNC (v24) 維持

## 動作確認
- `db-audit.html` において、現在の学習ログに基づいた「推奨配分」と「推奨理由」が正しく表示されることを確認。
- 実際の「今日の学習」を開始した際、従来の 24:6 配分が一切崩れずに維持されていることを確認（回帰テスト合格）。
- 各Viewでの回答操作と FSRS 更新が、本拡張の影響を受けずに正常動作し続けていることを確認。

## P7 (次フェーズ) への課題
- **自動適用の解禁**: 推奨配分を実際の Daily Session キュー生成に反映させるオプションの追加。
- **特化型カード統合**: NumberRecall 等を動的スロットとして組み込む。
- **長期最適化**: 1ヶ月単位の忘却曲線に基づいた、より深い配分チューニング。
