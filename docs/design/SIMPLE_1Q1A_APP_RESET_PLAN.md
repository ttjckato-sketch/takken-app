# SIMPLE 1Q1A App Reset Plan

**目的**: 宅建・賃貸管理士アプリを「分野別1問1答 + 問題専用解説 + 誤答反復」に戻す。  
**方針**: 新機能を増やさず、既存資産を最小導線に整理する。DB復旧、監査、runner、分析基盤を学習体験の中心に置かない。

## 1. 現状の問題点

現在のアプリは、本来の学習導線よりも周辺機能が前面に出ている。

| 問題 | 内容 | 学習者への影響 |
|---|---|---|
| 目的の分散 | ActiveRecall、MemoryRecall、進捗分析、DB復旧、監査runner、phase管理が並列に増えている | 何をすればよいアプリか分かりにくい |
| 画面の過密 | ダッシュボード、統計カード、カテゴリ進捗、MemoryRecall導線がトップに混在している | 分野を選んで問題を解くまでの距離が長い |
| DB/runner中心化 | 復旧runner、監査HTML、dry-runレポートが開発の主役になっている | 学習体験の改善が後回しになる |
| 解説の役割不明確 | `high_quality_input_units`、`question_explanations`、`choice_explanations`、prototype fallback が混在している | 誤答直後に何を表示するべきかが曖昧になる |
| 分析過多 | FSRS、詳細履歴、弱点分析、confusion pairs などが早く入りすぎている | 1問1答の完成前に設計が肥大化する |

軌道修正の基準は、学習者が「分野を選ぶ → 1問解く → すぐ解説を読む → 間違えた問題を繰り返す」だけで学習できることとする。

## 2. 新しい完成定義

完成品は以下を満たす。

| 項目 | 完成条件 |
|---|---|
| 分野選択 | 権利関係、宅建業法、法令上の制限、税その他、賃貸管理士を選べる |
| 1問1答 | 過去問由来の分解問題を1問ずつ解ける |
| 正誤判定 | ○ / × または選択肢回答後、すぐ正誤が分かる |
| 問題専用解説 | その問題に対応した解説が出る |
| 理由理解 | なぜ正しいか、なぜ誤りかが分かる |
| ひっかけ補修 | どこで間違えやすいかが分かる |
| 1行暗記 | 試験直前に使える短い記憶文がある |
| 誤答反復 | 間違えた問題だけ再出題できる |
| 分野集中 | 同じ分野を続けて反復できる |

完成判定は、監査レポート数やrunner数ではなく、上記の学習導線が壊れずに動くかで行う。

## 3. 必要機能

最小アプリに残す機能は以下に限定する。

| 機能 | 残す理由 | 扱い |
|---|---|---|
| `source_questions` | 問題本体の正本 | 必要 |
| `source_choices` | 1問1答化した選択肢・正誤の正本 | 必要 |
| `question_explanations` | 問題全体の専用解説 | 必要 |
| `choice_explanations` | 選択肢単位の最優先解説 | 必要 |
| `high_quality_input_units` | v30解説がない場合の教材fallback | 必要 |
| `ActiveRecall` | 1問1答の中心画面 | 必要 |
| 分野フィルタ | 分野別学習の入口 | 必要 |
| 誤答時の解説表示 | 本アプリの品質差分 | 必要 |
| 間違えた問題だけ再出題 | 反復学習の最小機能 | 必要 |

解説表示の優先順位は固定する。

1. `choice_explanations`
2. `question_explanations`
3. `high_quality_input_units`
4. 既存prototype fallback
5. 最終fallback

## 4. 保留機能

以下は削除ではなく保留する。1問1答の芯が完成するまで、トップ導線や通常学習導線に出さない。

| 機能 | 保留理由 | 再開条件 |
|---|---|---|
| MemoryRecall | 1問1答と目的が異なり、現時点では導線を複雑にする | 1問1答が安定し、暗記カード化の要件が明確になった後 |
| `memory_cards` | 解説表示とは別の教材単位 | MemoryRecall再開時 |
| `memory_card_progress` | FSRS/記憶状態管理は初期完成に不要 | 反復機能が不足すると判明した後 |
| `memory_study_events` | ActiveRecall履歴と混ぜると危険 | MemoryRecall再開時 |
| 詳細な学習履歴 | 最初は誤答再出題に必要な最小履歴で足りる | UX上必要になった後 |
| FSRS | 現段階では過剰 | 問題数・利用頻度が増えてから |
| `confusion_pairs` | 高度な弱点分析であり初期完成に不要 | 誤答データが十分溜まった後 |
| 高度な弱点分析 | ダッシュボード化を招く | 1問1答完了後に別計画で検討 |

## 5. 不要・停止機能

以下は当面停止する。既存コードを即削除する必要はないが、新規開発・UI露出・runner追加を止める。

| 停止対象 | 停止理由 |
|---|---|
| ダッシュボード | 分野選択より前面に出ると学習導線を壊す |
| 複雑な進捗グラフ | 初期完成には不要 |
| 学習イベント分析 | 実利用前に分析基盤を増やすのは早い |
| runner追加 | DB復旧・監査runnerが増えすぎて保守負荷が高い |
| 監査HTML追加 | 本番学習体験と関係しないページ増殖を止める |
| DB復旧runner増殖 | 正本origin復旧後は原則追加しない |
| 大量のphase管理 | 目的より手続きが肥大化する |
| 「学習エンジン」という抽象化 | まず1問1答の具体導線を完成させる |
| 本番公開前でないpublic/dist監査の過剰化 | リリース直前の確認に限定する |

## 6. 最小DB構成

最小構成は、問題、選択肢、解説、最小履歴に絞る。

| 区分 | テーブル | 用途 | 方針 |
|---|---|---|---|
| 問題 | `source_questions` | 問題文、カテゴリ、試験種別、問題ID | 正本として保持 |
| 選択肢 | `source_choices` | 1問1答文、正誤、選択肢ID | 上書き禁止。`is_statement_true` はAI推測で変更しない |
| 解説 | `choice_explanations` | 選択肢単位の最優先解説 | read-only表示に使う |
| 解説 | `question_explanations` | 問題単位のfallback解説 | read-only表示に使う |
| 解説fallback | `high_quality_input_units` | 論点教材fallback | v30解説がない場合のみ使う |
| 最小履歴 | `study_events` | 誤答再出題に必要な回答結果だけ | 分析用途へ拡張しない |

最小履歴で持つ情報は、当面以下だけで足りる。

| 項目 | 用途 |
|---|---|
| `question_id` / `choice_id` | 誤答問題の再出題 |
| `category` | 同じ分野を続ける |
| `answered_correct` | 間違えた問題だけ抽出 |
| `answered_at` | 直近の誤答を優先する |

以下は最小DB構成から外す。

| テーブル | 扱い |
|---|---|
| `memory_cards` | 保留 |
| `memory_card_progress` | 保留 |
| `memory_study_events` | 保留 |
| `confusion_pairs` | 保留 |
| `restoration_candidates` | 復旧用sidecar。通常学習導線では使わない |

## 7. 最小画面構成

画面は最大3つに限定する。

### A. 分野選択画面

役割は「どの分野を解くか」を選ぶだけにする。

表示するもの:

- 権利関係
- 宅建業法
- 法令上の制限
- 税その他
- 賃貸管理士
- 必要なら各分野の未回答数・誤答数だけ

表示しないもの:

- 詳細グラフ
- FSRS
- MemoryRecall導線
- 監査・復旧・runner導線

### B. 1問1答画面

役割は「1問解いて、すぐ正誤を見る」ことに限定する。

表示するもの:

- 問題文
- ○ / ×
- 回答ボタン
- 次の問題
- 同じ分野を続ける
- 間違えた問題だけ解く

表示しないもの:

- ダッシュボード
- 詳細統計
- 複数モード切替
- 学習イベント分析

### C. 解説画面

役割は「誤答直後に理解を補修する」ことに限定する。画面として分けても、1問1答画面内のパネルでもよい。

表示順:

1. 結論
2. 理由
3. この問題へのあてはめ
4. どこで間違えやすいか
5. 1行暗記
6. 比較ポイント

表示しないもの:

- 長い監査情報
- source refs の詳細一覧
- 生成プロセス
- 信頼度グラフ

## 8. 今後の開発優先順位

優先順位は以下に固定する。

| 優先 | 内容 | 完了条件 |
|---|---|---|
| 1 | 分野選択から1問1答へ直行できる導線にする | 分野を選ぶと即問題が出る |
| 2 | 誤答直後の解説表示を安定させる | choice/question/HQI/fallback の順に表示される |
| 3 | 間違えた問題だけ再出題する | 誤答履歴から再出題できる |
| 4 | 同じ分野を反復できる | 次問が同じ分野から出る |
| 5 | 保留機能を必要に応じて再評価する | 1問1答が完成してから判断する |

## 9. 次にやる実装タスク

次にやる実装タスクは最大2つに絞る。

### Task 1. トップ画面を分野選択に戻す

目的:

- 現在のダッシュボード中心の入口を、分野選択中心に戻す。

実装方針:

- 既存コードを大きく削除せず、通常導線ではダッシュボード・MemoryRecall・分析カードを出さない。
- 表示するカテゴリは5つに固定する。
- 分野を選ぶとActiveRecallに入る。

受入条件:

- 起動直後に分野選択が見える。
- ダッシュボードが主画面に出ない。
- MemoryRecall導線が主画面に出ない。

### Task 2. ActiveRecallを最小1問1答フローに整理する

目的:

- 問題、回答、正誤、解説、次問の流れを一本化する。

実装方針:

- 既存 `ActiveRecallView` と `explanationMatcher` を使う。
- 誤答時は `choice_explanations` → `question_explanations` → `high_quality_input_units` → fallback の順で表示する。
- 解説表示は「結論、理由、あてはめ、間違えやすいポイント、1行暗記、比較」に限定する。

受入条件:

- 誤答直後に解説が出る。
- 解説の読む順番が固定されている。
- v30解説がない問題でも既存fallbackが壊れない。
- DB書込は最小回答履歴のみで、`source_choices` と `is_statement_true` は変更しない。

## 10. 禁止事項

このリセット方針が完了するまで、以下を禁止する。

| 禁止事項 | 理由 |
|---|---|
| 新しいダッシュボード追加 | 芯が再び分析側へずれる |
| 新しい進捗分析追加 | 1問1答完成前は過剰 |
| 新しいrunner追加 | runner増殖を止める |
| 新しい監査HTML追加 | 学習体験に直結しない |
| DB復旧runner追加 | 正本復旧後は追加不要 |
| `source_choices` 上書き | 正誤正本を壊す危険がある |
| `is_statement_true` のAI推測変更 | 法令正誤の破壊につながる |
| `study_events` と `memory_study_events` の混在 | ActiveRecallとMemoryRecallの履歴が壊れる |
| `memory_cards` と `memory_card_progress` の混在 | 教材本体と記憶状態が混ざる |
| human_review_required データの自動投入 | 品質保証前の混入を防ぐ |
| package変更 | 軌道修正目的から外れる |
| commit / push / deploy | 方針確定と実装確認後に別タスクで行う |

## 11. 判定

**判定: A**

この計画は、アプリの目的を「分野別1問1答 + 良い解説 + 誤答反復」に戻すための最小構成を定義している。次の実装は、ダッシュボードや分析を増やすことではなく、分野選択とActiveRecallの導線を単純化することに限定する。
