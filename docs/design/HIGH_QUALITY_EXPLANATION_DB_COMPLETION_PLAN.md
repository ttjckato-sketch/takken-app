# HIGH QUALITY EXPLANATION DB COMPLETION PLAN

## 1. 完成の定義 (Standard of Completion)
アプリ内のすべての問題（1,524問）および選択肢（3,024件）に対して、**v30 高品質解説スキーマ**に準拠したデータが紐付いており、学習者が「なぜそうなるか」を秒速で理解できる状態。

## 2. 実装ロードマップ

### Phase 1: 基盤強化 (200問 / Coverage 13%)
- **優先順位**: 権利関係（制限行為能力者、意思表示、代理）、宅建業法（免許、35条書面、37条書面）。
- **タスク**:
  - 現存する 28件の `human_review` データを修正して投入。
  - 新規に 140件の解説を GPT-4o 等の高品質モデルで生成。
- **目標**: 学習開始直後の「最初の10問」ですべて高品質解説が出る状態。

### Phase 2: 全分野網羅 (800問 / Coverage 52%)
- **優先順位**: 法令上の制限（都市計画法、建築基準法）、税・その他。
- **手法**: AI生成 + 自動品質バリデーター（論理矛盾検知）。
- **目標**: 学習の半分以上が「高品質解説」で進行する状態。

### Phase 3: 全件完成 (1,524問 / Coverage 100%)
- **優先順位**: 難問・奇問・歴史的過去問。
- **手法**: Legacy 解説のリファクタリング生成。
- **目標**: fallback テンプレートの完全廃止。

## 3. 監査・投入フロー
1.  **AI生成**: v30 ポリシー（結論・理由・あてはめ）に基づき生成。
2.  **自動監査**: `label_conflict_suspected` (正誤矛盾) および `placeholder_check` を実行。
3.  **人間監査 (B判定)**: 重要な論点、または自動監査に引っかかったものを人間が確認。
4.  **正式投入**: `ProductionBatchLoader` を使用し、Atomic に投入。

## 4. 品質基準 (v30-A)
- **結論**: 「〇（正しい）」または「×（誤り）」の断定から始める。
- **理由**: 条文名・判例趣旨を明記する。
- **あてはめ**: 「AがBに売却した場合〜」といった具体名を使う。
- **暗記支援**: 語呂合わせ、または「1行暗記（ポイント）」を付与する。
