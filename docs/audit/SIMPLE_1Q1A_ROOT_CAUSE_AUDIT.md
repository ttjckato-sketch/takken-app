# SIMPLE 1Q1A ROOT CAUSE AUDIT (2026-05-12)

## 【現状判定】
A. 原因が具体的に特定され、次に直すべき場所が明確

## 【今のアプリが使えない主因】
1. **「ハリボテの高品質」ロジック**: `explanationBuilder.ts` 等でデータがない場合に「不動産取引の基本原則に基づきます」といった**汎用テンプレート（Boilerplate）を大量に差し込んでおり**、学習者が本当に知りたい「なぜこの問題が×なのか」に答えていない。
2. **データ格差の放置**: 高品質な v30 データはわずか 20 件程度（Batch 1）であり、母集団の 4,000 件以上の legacy データの多くは解説が空か、自動生成の質の低いものである。
3. **破壊的なデータロード**: セッション開始ごとに 45MB の `ULTIMATE_STUDY_DECK.json` を全件 clear & put しており、**ユーザー進捗の喪失リスクと、UIの致命的なラグ**を引き起こしている。
4. **責務のスパゲッティ化**: `ActiveRecallView` がデータ取得、判定、解説構築、FSRS管理のすべてを抱え込んでおり、UIの修正がロジックの破壊に直結しやすい。
5. **検証成果物のノイズ**: 100 ファイルを超える `docs/audit` や `tools/dev-pages` がプロジェクトを占拠しており、**「何が本番コードで、何が使い捨ての検証用か」の区別が学習アプリとして成立していない。**

## 【フォルダ構造】
- `src/components/learning`: 25+ ファイル。似た名前の View が乱立（`UnderstandingCardView` vs `ModernUnderstandingCardView`）。
- `src/utils`: 50+ ファイル。責務が細分化されすぎており、データ投入ロジックだけでも 5 種類以上存在する。
- `docs/audit` & `docs/generated`: 80+ ファイル。過去の全ステップのエビデンスが残っており、リポジトリの 8 割が「報告書」になっている。

## 【データ品質の実態】
| データ種別 | 推定件数 | 品質 / 実態 |
|---|---|---|
| `source_questions` | 1,500+ | 良好（過去問ベース） |
| `source_choices` | 3,000+ | 良好（過去問ベース） |
| `v30_explanations` | **~20** | **極めて良好だが圧倒的に不足** |
| `legacy_cards` | 4,000+ | **解説が断片的、または自動生成で誤りを含む** |
| `fallback_templates` | N/A | **「もっともらしいが意味のない」長文で水増し** |

## 【機能不全】
- **FSRS スケジュール**: データロード時に clear されるため、事実上機能していない。
- **分野別進捗**: 常に「未学習」に戻る、またはカウントが不正確。
- **補修ユニット (HQI)**: 20 件しかないため、ほとんどの問題で「整備中」または汎用解説が出る。

## 【原因分類】
- **原因1: A判定基準のズレ**: 「ビルドが通る」「データが入る」ことを優先し、「学習者が理解できるか」という教材品質の監査が抜けていた。
- **原因2: 複雑なDB移行**: 26 テーブルにおよぶ IndexedDB スキーマが、同期や不整合の温床となっている。
- **原因3: プロトタイプからプロダクトへの転換失敗**: 検証用の runner や fallback をそのまま残したまま「リリース assets」としてしまった。

## 【削る候補】
- **即停止**: 全件 clear 後の再ロード (`autoDataLoader.ts` の clear 処理)。
- **保留**: `MemoryRecall`, `TrapRecall` 等の、データが揃っていないサブモード。
- **隔離**: 過去の `docs/audit`, `docs/generated` は別フォルダ、またはアーカイブへ。

## 【結論】
今のアプリは**「巨大な過去の残骸の上に、ごく少数の新しい宝石（v30）が乗っている」**状態です。
宝石が少なすぎるため、ユーザーは常に残骸（fallback）を見せられ、アプリが壊れている、あるいは質が低いと感じます。

### 最優先で直すべき3つ:
1.  **データの非破壊化**: `autoDataLoader` を「差分更新（または初回のみ）」に変更し、学習データを保護する。
2.  **fallback の誠実化**: 意味のない汎用テンプレートを廃止し、データがない場合は「解説未整備」と明示するか、Raw Trace のみを表示する。
3.  **ランタイムの軽量化**: 不要な旧 View と Utils を削除し、ActiveRecall 1 本に絞る。

### 次の修正タスク:
1.  `src/utils/autoDataLoader.ts` の `clear()` 処理を廃止し、IndexedDB の永続性を確保する。
2.  `src/utils/explanationBuilder.ts` の汎用テキスト（Boilerplate）を削除し、v30 データの存在を際立たせる。
