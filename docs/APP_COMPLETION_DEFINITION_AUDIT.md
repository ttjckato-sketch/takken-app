# 宅建・賃貸管理士学習アプリ 完成定義v2 監査レポート (2026-05-03)

## 1. 監査の背景と目的
復元PoCにより23件の教材復元に成功したことを受け、アプリ本体が「完成定義v2」に掲げる「迷わず回せる学習サイクル」にどの程度到達しているかを客観的に評価した。

## 2. 完成定義v2 到達状況 (2026-05-03時点)

### 【ActiveRecall (演習)】
- **現状**: 1024件の安全母集団が正常に稼働。Playwright実機テストにより回答イベントの保存（+1）も実測済み。
- **判定**: **ほぼ完成 (A)**。
- **課題**: 2621件の破損カードが未だ除外されている点。

### 【RepairPreview / Input Unit (補修)】
- **現状**: 23件の代表的なInput Unitプロトタイプが存在。誤答時の表示ロジックは実装済みだが、カテゴリ分類が一部 null であり、タグスコアリングの精度に改善の余地。
- **判定**: **ベータ版 (B)**。
- **課題**: Input Unit のカテゴリ正規化と、カバー範囲の拡大（現在は主要論点のみ）。

### 【MemoryRecall (復習)】
- **現状**: FSRSアルゴリズムによる復習スケジューリング、10Qセッションの起動、復元由来カードの表示まで一貫して動作。
- **判定**: **ほぼ完成 (A)**。
- **課題**: 復元された肢テキストと解説の質が、AI補完プロセスに依存する点。

### 【弱点分析・ダッシュボード (分析)】
- **現状**: 直近の学習履歴がない初期状態では、弱点表示が機能しない「冷たいスタート」問題を確認。優先タグの推薦ロジックは存在するが、蓄積データ不足への配慮が必要。
- **判定**: **機能実装済 (B)**。
- **課題**: 学習初期ユーザーに対する「最初の導線」の親切さ（ガイダンス表示など）。

### 【比較学習 (比較)】
- **現状**: `confusion_pairs` テーブルおよびデータ構造は存在するが、アプリ本体に独立した「比較学習モード」の専用導線が未実装。Input Unit内の比較表に依存している。
- **判定**: **未完成 (C)**。
- **課題**: 「混同しやすい論点」を一覧・集中学習できるUIの追加。

## 3. 復元PoCの位置づけ
復元PoCは「教材の質と量を自律的に改善する補助工程」として定義通り機能している。現在は43件中23件が昇格済みであり、1024件の安全母集団を破壊せずに本流へデータを戻すパスは完全に確立された。

## 4. 完成までの残作業トップ5 (優先順)
1. **比較学習専用UIの実装**: 混同論点（confusion_pairs）を直接学習できるカードビューの追加。
2. **Input Unit のカテゴリ正規化**: null となっている prototype のカテゴリを整備し、誤答時の補修精度を向上させる。
3. **初期学習者向けガイダンス**: 履歴が0件の際、ダッシュボードに「まず演習から始めよう」等の動的な案内を表示。
4. **2621件の本格復元スケーリング**: PoCフローを自動化し、安全母集団を数千件規模へ拡張。
5. **タグスコアリングの再調整**: 復元された KU が既存のタグベースの学習推薦ロジックに及ぼす影響を微調整。

## 5. 総合判定
**B+ (完成間近)**
「演習・補修・復習」のメインループは実測レベルで完結している。
「比較・分析」の UI 的な統合を完了させることが、v2完成への最短距離である。
