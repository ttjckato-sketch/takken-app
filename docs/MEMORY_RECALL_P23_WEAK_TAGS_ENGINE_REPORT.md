# MemoryRecall P23 Weak Tags Engine Report

## 概要
学習履歴（study_events）から学習者の苦手分野を自動算出する `weak_tags` 算出エンジンを最小実装した。
直近 100件の学習結果に基づき、タグ（分野）ごとにスコアリングを行い、最も対策が必要な Top 5 を特定・監査可能にした。

## 実施内容
- **weak_tags 算出エンジンの実装**:
  - `analytics.ts` に `calculateWeakTagsMetrics` を追加。
  - 誤答（+3）、Rating 1（+3）、Rating 2（+2）、FSRS 難易度 7以上（+2）の重み付けでスコアを算出。
  - イベントにタグがない場合、`understanding_cards` または `memory_cards` から動的にタグを解決するロジックを実装。
- **Metadata キャッシュの導入**:
  - 算出結果を `metadata` ストアの `weak_tags_cache` キーに保存し、再計算負荷を軽減。
- **監査 UI の強化**:
  - `db-audit.html` に「📉 P23 苦手分野 (Weak Tags) Top 5」および算出メタデータの表示セクションを追加。
  - ダウンロード可能な報告フォーマットにも Top 5 タグを含めるよう拡張。

## 結果 (実測値)
- **集計ウィンドウ**: 直近 100件のイベント
- **算出項目**: スコア、誤答数、Rating 1/2数、出現件数
- **Build 結果**: PASS
- **Schema 整合性**: FULL_SYNC (v24) 維持

## 動作確認
- `db-audit.html` において、実際の学習ログから「宅建業法」「重要事項説明」などの苦手タグが正しく抽出され、スコアリングされることを確認。
- 30問標準・35問集中の各モード、および `auto_apply` 安全制約が、エンジン実装後も正常に機能することを回帰確認。

## P24 (次フェーズ) への課題

- 弱点タグが付与されたカードを優先出題する配分エンジンの設計。
