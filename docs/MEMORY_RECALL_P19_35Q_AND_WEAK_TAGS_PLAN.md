# MemoryRecall P19 35Q & Weak Tags Plan

## 1. 背景
MemoryRecall P18 において、5モード（Active, Memory, Number, Trap, Comparison）を統合した 1日30問セッションの安定稼働を物理監査した。
P19 では、さらなる学習効果の向上を目指し、セッション総数の 35問への拡張、および苦手分野を可視化する `weak_tags` グラフの導入を設計する。

## 2. 目的
- **35問化**: ActiveRecall（演習）の量を 24問に復帰させつつ、周辺 4モードの露出を維持・強化する。
- **weak_tags グラフ**: 学習履歴（study_events）から「間違いの多い分野」を自動抽出し、学習者に自覚を促す。
- **UI 磨き込み**: Dashboard を「管理者向け」から「学習者向け」へと最適化し、モチベーション維持を図る。

## 3. 30問維持案 vs 35問拡張案
| 項目 | 現状 (30Q) | P20目標 (35Q) | メリット |
| :--- | :--- | :--- | :--- |
| **Active** | 22 | **24** | 演習の絶対量を確保。 |
| **Memory** | 3 | **4** | 語彙力の維持。 |
| **Number** | 2 | **3** | 数字ひっかけへの耐性強化。 |
| **Trap** | 2 | **2** | ひっかけパターンの習得。 |
| **Comparison** | 1 | **2** | 構造的理解の深化。 |
| **合計** | 30 | **35** | 1セッションの密度を極大化。 |

## 4. weak_tags グラフの設計
- **指標**: 過去 100件のイベントまたは直近 7日間の「Rating 1(Again) / 2(Hard) の比率」を tag ごとに算出。
- **表示**: Dashboard トップに「要注意タグ Top 5」を棒グラフまたはリストで表示。
- **FSRS 連携**: 苦手タグが付与されたカードの `difficulty` パラメータを監査し、優先的に出題するロジック（P22以降）への足掛かりとする。

## 5. 安全制約と整合性
### auto_apply (自動適用)
- P19/P20 時点では、35問配分は「固定拡張」として扱い、auto_apply は引き続き Active/Memory の相対比率のみを調整する。
### manual_pending (手動予約)
- `total_count: 35` の予約をサポートするか検討（P20 実装時に決定）。
### rollback (ロールバック)
- 35問化による負荷過多が確認された場合、即座に P18 の 30問（22:3:2:2:1）へ回帰する。

## 6. db-audit 監査項目
- `session_total_count`: 30 or 35。
- `weak_tags_calculation_integrity`: 苦手タグ算出ロジックの正当性。
- `performance_impact`: 35問化による App 動作への影響。

## 7. Acceptance Criteria (AC)
- **AC-001**: P18 の 5モード安定状態を維持。
- **AC-002**: 35問配分（24:4:3:2:2）の論理的整合性の確認。
- **AC-003**: `weak_tags` 算出アルゴリズムの定義。
- **AC-004**: Dashboard UI 刷新案の策定。
- **AC-005**: Build PASS / Schema v24 維持。
