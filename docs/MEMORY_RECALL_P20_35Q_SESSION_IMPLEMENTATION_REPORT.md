# MemoryRecall P20 35Q Session Implementation Report

## 概要
上位学習者および直前期向けの集中モードとして `35問セッション` を最小実装した。
P18 で凍結した 30問標準モード（22:3:2:2:1）をデフォルトとして維持しつつ、手動で 35問（24:4:3:2:2）を選択・開始できる仕組みを構築した。

## 実施内容
- **配分エンジンの拡張**:
  - `buildDailyStudySessionQueue` に `variant` オプションを追加し、`35q` 指定時に演習量を 24問に引き上げた 5モード配分を実現。
  - 35問モードは `auto_apply` (自動配分調整) の対象外とし、固定の黄金比率で提供。
- **セッション追跡の強化**:
  - `startStudySession` および `completeStudySession` を更新し、`session_variant` (30q/35q) を正確に記録。
  - セッション総数 35問の保存を物理確認。
- **Dashboard 入口の追加**:
  - メイン画面に「集中セッション (Zapアイコン)」ボタンを追加。
  - モバイル環境でも視認可能な 35Q ボタンを配置。
- **監査 UI の更新**:
  - `db-audit.html` に 35Q 集中セッションの管制セクションを追加し、設計値（24:4:3:2:2）の妥当性を監視可能にした。

## 結果 (実測値)
- **35Q 配分比率**: Active 24 : Memory 4 : Number 3 : Trap 2 : Comparison 2
- **30Q 標準比率**: Active 22 : Memory 3 : Number 2 : Trap 2 : Comparison 1 (維持)
- **Build 結果**: PASS
- **Schema 整合性**: FULL_SYNC (v24) 維持

## 動作確認 (物理監査完了)
- 35問セッションにおいて、Active 24問を含む全 5モードが設計通りに出題されることを E2E 実測。
- 30問標準セッションの配分および自動適用ガードが、新機能追加後も壊れていないことを回帰確認。

## P21 (次フェーズ) への課題
- `weak_tags` (苦手分野) の算出ロジック実装と Dashboard グラフ表示。
- 35問セッションの長期継続率の監視。
- 学習者向け UI のさらなる磨き込み（管理者指標の完全隠蔽）。

## 判定
**A. MemoryRecall P20 実装OK**
30問の安定性を損なうことなく、より高い学習密度を求める学習者向けの「集中モード」を安全に導入した。
