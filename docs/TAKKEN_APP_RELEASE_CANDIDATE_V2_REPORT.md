# 宅建・賃貸管理士学習アプリ リリース候補v2 固定レポート (2026-05-03)

## 1. 完成定義v2 への到達点
「問題演習、誤答補修、記憶カード復習、弱点分析、比較学習を1つの流れで迷わず回せる状態」に対し、主要機能がすべてMVPレベルで動作することを確認し、本バージョンをリリース候補 (RC1) として固定した。

## 2. 主要要素の判定 (A/B/C)
- **ActiveRecall (演習)**: **A**
  - 1024件の安全母集団が稼働中。実カード回答イベントの保存も実証済み。
- **RepairPreview / Input Unit (補修)**: **B+**
  - 主要論点（業法中心）の構造化インプットが機能。カテゴリ正規化が今後の改善点。
- **MemoryRecall (復習)**: **A**
  - FSRSアルゴリズムによるスケジューリングと10Qセッションが完結。復元データも統合済み。
- **弱点分析 (分析)**: **B**
  - 学習ログに基づく動的な優先タグ推薦が稼働。初期ユーザーへのガイダンスに改善余地。
- **比較学習 (比較)**: **B (NEW)**
  - `ComparisonLearningView` の実装により、混同論点の対比学習が可能となった。

## 3. 復元PoCの役割整理
復元PoC（現在23件昇格済）は、アプリ本体の機能ではなく「教材品質の自律修復および量的拡張のための補助工程」である。サイドカーテーブル方式により、本流の1024件安全母集団を破壊せずに教材を増強できるパスが確立されている。

## 4. リリース候補としてのステータス
**RELEASE CANDIDATE (RC1) 承認**
コアとなる「学習の正のループ」は完成しており、実用可能な状態にある。

## 5. 残作業の分類
### 【必須 (Must Fix) - 完了済】
- **Input Unit カテゴリ正規化 (2026-05-03)**: `TAKKEN_PROTOTYPE_UNITS` の全件に標準カテゴリを付与。Validator および Matcher も更新済み。

### 【改善 (Improvements) - 完了済】
- **初期学習者用案内 (2026-05-03)**: 学習履歴が0件の際、ダッシュボードに「最短で合格を手に入れる４ステップ」を動的に表示。演習・補修・復習・比較のサイクルを視覚化。

### 【改善 (Remaining)】
- **比較学習データの拡充**: Input Unit 以外からも `confusion_pairs` を吸い上げるロジックの追加。

### 【拡張 (Expansions)】
- **2621件の本格復元**: PoCフローを数千件規模へスケーリングし、網羅性を極大化。
- **全件統合**: 復元された `source_choices` を本体の `correct_patterns` へマージ。

## 6. 次にやること
1. **リリース候補の最終パッケージ化**: 確定したシードデータと動的ガイダンス UI を含む v2.0-rc1 の最終バイナリ（dist 成果物）を固定し、配布可能な状態にする。
2. **2621件の本格復元設計 (Next Version)**: 本PoCの成果を元に、数千件規模の破損カードを自動修復する次世代パイプラインの要件を定義する。
