# MemoryRecall P22 Weak Tags & Dashboard Plan

## 1. 背景
MemoryRecall P21 において、35問集中セッション（Active 強化配分）の安定稼働を物理監査した。
P22 では、蓄積された学習履歴（study_events）を活用し、学習者の苦手分野を可視化する `weak_tags` グラフの導入、および「次に何をすべきか」を提示する Dashboard 強化を設計する。

## 2. 目的
- **弱点の自己認識**: 誤答や低評価評価（Rating 1/2）が多い分野を特定し、学習者に自覚を促す。
- **効率的な復習の誘導**: 苦手タグに基づき、最適な学習モード（Number/Trap/Comparison）を推奨する。
- **モチベーションの維持**: 漠然とした不安を「具体的な課題」に変換し、学習継続率を高める。

## 3. weak_tags 算出ロジックの設計
### データ源
- **集計対象**: 直近 100件の `study_events` をスライディングウィンドウとして使用。
- **利用フィールド**: `mode`, `answered_correct`, `rating`, `tags`, `category`, `created_at`。

### スコアリングアルゴリズム (Weakness Score)
各タグ（分野）に対し、以下の重み付けで累積スコアを算出する。
| 指標 | 重み | 理由 |
| :--- | :--- | :--- |
| **answered_correct = false** | **+3** | 明確な知識の欠落。 |
| **Rating 1 (Again)** | **+3** | 忘却または未習得。 |
| **Rating 2 (Hard)** | **+2** | 定着が不安定。 |
| **FSRS Difficulty > 7** | **+2** | 構造的に難しい論点（FSRS 内部値）。 |
| **Overdue (未完了期限切れ)** | **+1** | 復習が滞っている分野。 |

### 算出処理
- `analytics.ts` に `calculateWeakTagsMetrics()` を実装。
- タグごとのスコアを算出し、降順でソートして Top 5 を抽出。

## 4. Dashboard 表示方針 (学習者向け)
### 可視化
- **グラフ形式**: シンプルな水平棒グラフ、または Top 5 タグのバッジリスト。
- **表示項目**: タグ名、苦手度（スコア）、その分野で最近間違えた問題へのリンク（任意）。

### 推奨アクション (Next Action)
算出された苦手傾向に基づき、Dashboard トップにアドバイスを表示する。
- **数字・期間が苦手**: 「NumberRecall で数字を固めましょう」
- **ひっかけ・例外が苦手**: 「TrapRecall で罠回避を特訓しましょう」
- **概念の混同が目立つ**: 「ComparisonRecall で制度の違いを整理しましょう」
- **全体的に正答率が低い**: 「30問標準モードで基礎を固めましょう」

## 5. db-audit 監査方針 (管理者向け)
- 全タグのスコアリング詳細（誤答数、低評価数、Difficulty 平均）を表示。
- `session_variant`（30q/35q）別の弱点傾向の比較。
- タグ解決失敗（Lookup Failure）のログ監視。

## 6. 技術整合性と整合性
- **Schema v24 維持**: 新テーブルは作成せず、算出はメモリ上で行う。必要に応じて `metadata` にキャッシュ。
- **自動適用への影響**: P22 時点では表示のみ。将来的に `weak_tags` を配分比率に反映させるための布石とする。

## 7. Acceptance Criteria (AC)
- **AC-001**: 既存の 5モード（Active/Memory/Number/Trap/Comparison）の安定稼働を維持。
- **AC-002**: 直近 100件のイベントから正確に苦手タグを抽出するロジックの定義。
- **AC-003**: 学習者向け Dashboard への「苦手 Top 5」表示案の策定。
- **AC-004**: スコアに基づく推奨アクション生成エンジンの設計。
- **AC-005**: Build PASS / Schema v24 維持。
