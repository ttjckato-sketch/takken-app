# MemoryRecall P24 Dashboard Weak Tags Integration Report

## 概要
P23 で実装した `weak_tags` 算出エンジンをメインダッシュボードに統合し、学習者が自身の弱点と次の一手をデータに基づいて把握できる機能を最小実装した。
「管理者向けの監査」から「学習者向けのフィードバック」へのシフトを図り、学習の質の向上を実現した。

## 実施内容
- **Dashboard への視覚的統合**:
  - `App.tsx` を更新し、トップ画面に「苦手分野 Top 5」カードを新設。
  - スコア、誤答数、Rating 1/2 の累積に基づき、直感的に弱点を把握できる UI を構築。
- **インテリジェント推奨アクションの実装**:
  - `analytics.ts` の `getStudyDashboard` を拡張し、算出された苦手タグに基づき「次のおすすめ学習アクション」を自動生成。
  - 数字が苦手なら `NumberRecall`、ひっかけなら `TrapRecall`、混同なら `ComparisonRecall` を優先的に提案するロジックを搭載。
- **UI/UX の磨き込み**:
  - 理由生成ロジックを追加（例: 「直近で誤答が多い分野です」）。
  - 軽量なインジケータとアイコン（TrendingDown, Target, Compass）を使用し、P10 以降の最小デザインを維持。
  - 管理者向けの生スコアなどの詳細は Dashboard から隠蔽し、`db-audit.html` に集約。

## 結果 (実測値)
- **Top 5 タグ表示**: 正常 (テストデータに基づく宅建業法、重要事項説明等の表示を確認)
- **推奨アクション生成**: 正常 (苦手分野の特性に応じた適切なアドバイスを確認)
- **Build 結果**: PASS
- **Schema 整合性**: FULL_SYNC (v24) 維持

## 動作確認
- 苦手 Top 5 カードにおいて、各タグの理由（誤答数等）が正確に提示されることを物理監査完了 (Dashboard 苦手 Top 5 および推奨アクションの表示を実測)。
- ダッシュボードの 30問標準・35問集中ボタンの配置および動作が、表示追加後も壊れていないことを回帰確認済み (30問/35問各モードの正常動作を確認)。
- `auto_apply` がダッシュボードの表示ロジックと干渉せず、安全に独立して機能していることを確認。

## P25 (次フェーズ) への課題
- P24 の Freeze 監査。
- 弱点タグが付与されたカードを優先出題する「苦手克服モード」の配分エンジン統合。
- 集計ウィンドウ（直近100件）における時間減衰（Recency Decay）の検討。

## 判定
**A. MemoryRecall P24 実装OK**
データの蓄積（P23）から価値の提示（P24）へと機能が繋がり、アプリとしての学習体験が一段階進化した。
